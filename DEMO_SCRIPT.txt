[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the underlying flow that makes this work.
(pause)
Every request starts at the Security Checkpoint, which performs PII scrubbing and prompt injection detection before any other node runs.
Once passed, safe inputs are routed to the orchestrator_agent.
The orchestrator_agent is the brains of our multi-agent system, and it uses AgentTools to delegate tasks.
It sends a request to the competitor_analyzer sub-agent, which calls the MCP get_competitor_prices tool to query competitor database tables.
Simultaneously, the elasticity_calculator sub-agent is triggered, calling the MCP get_product_sales_history tool to fetch sales volumes over time and estimate demand elasticity.
Both sub-agents report back, and the orchestrator recommends a new price.
This passes to the Decision Gate, which enforces our domain rules.
If the price change is under 10%, it is auto-approved and goes straight to the pricing_executor to call the database update tool.
But if it is a major price change over 10%, it routes to the Product Manager Approval HITL node, pausing the workflow until a human gives the green light.
